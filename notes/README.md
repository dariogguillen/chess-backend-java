# Notes

This folder contains a learning note for every feature shipped to this
project. Each note explains what was built, the Java and Spring Boot
concepts involved, the decisions taken, and how the patterns compare
to similar work in other ecosystems (Scala/Typelevel, Node).

The notes are deliberately written for a reader who is **learning Java
and Spring Boot from a different background**. They are not generic
tutorials. They walk through *this* code in *this* repo, with the
trade-offs and the context.

If you are reviewing this repository as a recruiter or hiring manager,
these notes are intentional — they demonstrate that every feature was
implemented with understanding, not by generation alone.

## Structure

```
notes/
├── README.md            # You are here
├── _template.md         # Template used for every feature note
├── 01-health-check.md   # One note per feature, prefixed by priority
├── 02-domain-models.md
├── ...
└── topics/              # Cross-cutting topics referenced from notes
    ├── spring-di.md
    ├── jpa-vs-functional-persistence.md
    ├── records-vs-case-classes.md
    └── ...
```

## Conventions

- Filename: `NN-<feature-id>.md`, with `NN` being the zero-padded
  priority from `feature_list.json`.
- Always written in English, even when the conversation around the
  feature happened in another language. The repo is in English.
- First-person allowed, but not required.
- Honest about gotchas and false starts. A note that says *"I first
  tried X and it didn't work because Y"* is more useful than one that
  pretends the path was straight.

## Workflow

The notes are generated by the `implementer` sub-agent as part of every
feature. See `.claude/agents/implementer.md` and
`.claude/agents/reviewer.md`. The reviewer rejects features whose note
is missing or empty.
